SKILL: db_contract_script_runner
Execute pre-approved, versioned contract SQL/shell scripts with checksum verification, approval gating, rollback, and full audit logging.
risk: high context: L3 requires_approval: true v2.1.0
Execution Steps
1
Validate Inputs
Assert script_name, target_database in registry, approval_ticket valid for production.
↓
2
Fetch Script from S3 + Verify Checksum
Download from s3://company-skill-library-prod/skills/.../contract_scripts/. SHA-256 must match L1 manifest. Mismatch → security incident, abort.
↓
3
Pre-flight Connection Check
psql SELECT 1 with 5s timeout. Abort with CONNECTION_FAILED if unreachable.
↓
4
If dry_run=true → Impact Report
Wrap in BEGIN/ROLLBACK. Capture estimated row counts, lock types, DDL changes. Return dry_run_report and exit.
↓
5
Execute in Transaction
BEGIN with statement_timeout and lock_timeout. On any error → ROLLBACK → execute paired rollback script if enabled.
↓
6
Post-execution Validation
Run *_validate.sql if present in S3. Assert validation_result.passed == true.
↓
7
Audit Log + Notify
Write structured JSON audit record to audit_log_store. Post summary to notify_channel if configured.

SQLVersion Sequence Guard
-- Step 1: Ensure sequential version ordering
SELECT version FROM schema_version
ORDER BY applied_at DESC LIMIT 1;
-- current=3.3.2, requested=3.4.0 → ALLOWED
-- current=3.3.2, requested=3.5.0 → REJECTED (gap)

-- Step 2: Idempotency check
SELECT 1 FROM schema_version
WHERE version = '3.4.0';
-- Row exists → migration already applied → no-op, return success

-- Step 4: Execute in transaction
BEGIN;
  -- [migration SQL injected here]
  INSERT INTO schema_version (version, applied_at, applied_by, checksum)
  VALUES ('3.4.0', NOW(), 'dba_agent', '<checksum>');
COMMIT;

SKILL: db_health_report
Read-only health assessment covering bloat, indexes, replication lag, lock contention, connection pool utilisation, and slow queries. Green/Yellow/Red per metric.
risk: low context: L2 read-only v1.4.0
SQLBloat Detection Query
SELECT
    schemaname, tablename,
    pg_size_pretty(pg_total_relation_size(
        schemaname||'.'||tablename)) AS total_size,
    ROUND(100 * (1 - (n_live_tup::numeric /
        NULLIF(n_live_tup + n_dead_tup, 0))), 2) AS dead_tuple_pct,
    last_autovacuum, last_autoanalyze
FROM pg_stat_user_tables
WHERE n_live_tup + n_dead_tup > 10000
ORDER BY dead_tuple_pct DESC LIMIT 20;

add_indexes_orders_v1.sql
Contract script: adds 4 concurrent non-locking indexes to the orders schema to resolve slow query findings from the health report.
migration: V3.4.0 CHANGE-2024-0847 CONCURRENT — no table lock requires approval
SQLadd_indexes_orders_v1.sql · v1.0.0
-- ============================================================
-- CONTRACT SCRIPT: add_indexes_orders_v1.sql · V3.4.0
-- Approval: CHANGE-2024-0847 | Risk: Medium
-- Rollback: add_indexes_orders_v1_rollback.sql
-- ============================================================

SET statement_timeout = '0';   -- Disable for CONCURRENT builds
SET lock_timeout      = '5s';
SET work_mem          = '256MB';

-- Index 1: Customer lookup with status filter
CREATE INDEX CONCURRENTLY IF NOT EXISTS
    idx_orders_customer_status
ON orders (customer_id, status)
WHERE status NOT IN ('archived', 'cancelled');

SELECT pg_sleep(2);   -- Reduce I/O pressure between builds

-- Index 2: Date-range queries with status
CREATE INDEX CONCURRENTLY IF NOT EXISTS
    idx_orders_created_at_status
ON orders (created_at DESC, status)
INCLUDE (customer_id, total_amount);

SELECT pg_sleep(2);

-- Index 3: order_items FK lookup
CREATE INDEX CONCURRENTLY IF NOT EXISTS
    idx_order_items_order_id
ON order_items (order_id)
INCLUDE (product_id, quantity, unit_price);

SELECT pg_sleep(2);

-- Index 4: Product sales aggregation
CREATE INDEX CONCURRENTLY IF NOT EXISTS
    idx_order_items_product_agg
ON order_items (product_id, created_at DESC)
INCLUDE (quantity, unit_price);

-- Refresh statistics post-build
ANALYZE orders;
ANALYZE order_items;

-- Record migration (idempotency guard)
INSERT INTO schema_version
    (version, description, applied_at, applied_by, script_name)
VALUES ('3.4.0', 'add_indexes_orders_v1', NOW(),
        current_user, 'add_indexes_orders_v1.sql')
ON CONFLICT (version) DO NOTHING;

add_indexes_orders_v1_rollback.sql
Paired rollback script for V3.4.0. Drops all four indexes concurrently and removes the schema_version entry. Safe to run partially applied migrations.
paired with: V3.4.0 IF EXISTS guards CONCURRENT drops
SQLadd_indexes_orders_v1_rollback.sql
-- Drop in reverse order of creation (IF EXISTS = safe on partial apply)
SET lock_timeout = '5s';

DROP INDEX CONCURRENTLY IF EXISTS idx_order_items_product_agg;
SELECT pg_sleep(1);

DROP INDEX CONCURRENTLY IF EXISTS idx_order_items_order_id;
SELECT pg_sleep(1);

DROP INDEX CONCURRENTLY IF EXISTS idx_orders_created_at_status;
SELECT pg_sleep(1);

DROP INDEX CONCURRENTLY IF EXISTS idx_orders_customer_status;

-- Refresh planner stats
ANALYZE orders;
ANALYZE order_items;

-- Remove schema_version entry
DELETE FROM schema_version
WHERE version = '3.4.0'
  AND script_name = 'add_indexes_orders_v1.sql';

add_indexes_orders_v1_validate.sql
Post-execution validation query. Returns a single JSON result; orchestrator checks result->>'passed' = 'true' before marking the execution as successful.
returns JSON orchestrator-parsed read-only
SQLadd_indexes_orders_v1_validate.sql
WITH index_check AS (
    SELECT
        indexname,
        CASE WHEN indisvalid THEN 'valid' ELSE 'INVALID' END AS index_status
    FROM pg_indexes
    JOIN pg_class  ON pg_class.relname  = pg_indexes.indexname
    JOIN pg_index  ON pg_index.indexrelid = pg_class.oid
    WHERE indexname IN (
        'idx_orders_customer_status',
        'idx_orders_created_at_status',
        'idx_order_items_order_id',
        'idx_order_items_product_agg'
    )
),
version_check AS (
    SELECT COUNT(*) AS cnt
    FROM schema_version
    WHERE version = '3.4.0'
)
SELECT json_build_object(
    'passed',           (SELECT COUNT(*) = 4
                          AND bool_and(index_status = 'valid')
                          FROM index_check)
                         AND (SELECT cnt = 1 FROM version_check),
    'indexes_found',     (SELECT COUNT(*) FROM index_check),
    'indexes_valid',     (SELECT COUNT(*) FROM index_check
                          WHERE index_status = 'valid'),
    'version_recorded',  (SELECT cnt FROM version_check),
    'errors',            (SELECT COALESCE(json_agg(
                              indexname || ': ' || index_status),
                              '[]'::json)
                          FROM index_check
                          WHERE index_status != 'valid')
) AS result;
